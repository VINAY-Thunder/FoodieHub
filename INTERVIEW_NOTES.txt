[Interview Q&A]
Q: "How did you structure controllers and services?"
A: I followed the standard 3-layer architecture: Controllers handle HTTP requests and validation, Services contain the business logic and orchestrate data flow, and Repositories handle database persistence via Spring Data JPA. This separation of concerns ensures that the business logic is decoupled from both the API and the database.

Q: "How did you handle validation and errors?"
A: I used '@Valid' and Jakarta Validation annotations on DTOs to catch invalid data early at the controller level. For errors, I implemented a '@RestControllerAdvice' class to globally catch custom exceptions and return consistent error messages with appropriate HTTP status codes.

Q: "How would you add authentication later?"
A: I would integrate Spring Security to handle JWT or Session-based authentication. This would involve adding a filter chain to secure endpoints, creating a UserDetails service, and ensuring that sensitive operations like order deletion are restricted to authorized users only.

Q: "How did you test your code?"
A: I focused on ensuring the build process was successful and verifying the logic through manual API walkthroughs using tools like Postman. I also ensured that custom methods in my repositories were correctly derived from Spring Data JPA naming conventions for reliable data retrieval.
